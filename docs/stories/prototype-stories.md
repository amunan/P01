# 原型故事包选型

> 版本：v0.1 | 状态：初版定稿

## 1. 选型原则

需要两个题材差异性大的故事包，验证 Story Package 协议的通用性。

| 维度 | 故事包 A | 故事包 B |
|------|---------|---------|
| 类型 | 政治/关系/谋略 | 调查/悬疑/事件推进 |
| 交互重心 | 多角色关系博弈、说服、结盟 | 线索收集、推理、审讯 |
| 验证重点 | 关系系统、记忆衰退、NPC 性格一致性 | 事件触发、知识边界、单线推进 |
| 角色数量 | 6-8 个 | 3-5 个 |
| 章节规模 | 3-5 章 | 2-3 章（单案件） |

## 2. 故事包 A：三国演义 · 赤壁弧线

### 基本信息

| 项目 | 内容 |
|------|------|
| ID | `three-kingdoms-chibi` |
| 题材 | 历史政治博弈 |
| 原作 | 《三国演义》（罗贯中），公有领域 |
| 截取范围 | 三顾茅庐 → 联吴抗曹 → 赤壁之战 |
| 玩家角色 | 刘备麾下无名谋士 |
| 预计时长 | 2-4 小时 |

### 为什么选这个

- **对话密度极高**：政治博弈、军事谋略大量依赖对话和辩论
- **角色辨识度极强**：曹操的雄才、诸葛亮的智慧、周瑜的骄傲——每个角色说话方式鲜明
- **主线里程碑清晰**：历史事件本身就是天然的里程碑节点
- **无版权风险**：古典名著
- **多势力多角色**：适合验证关系系统和记忆衰退

### 章节规划

| 章 | 标题 | 核心事件 | 里程碑 |
|----|------|---------|--------|
| 1 | 三顾茅庐 | 刘备三访诸葛亮 | 隆中对 |
| 2 | 联吴之议 | 诸葛亮出使东吴 | 舌战群儒、孙刘联盟成立 |
| 3 | 赤壁风云 | 战前准备与博弈 | 草船借箭、庞统献连环计 |
| 4 | 决战赤壁 | 火烧赤壁 | 赤壁之战、曹操败退 |
| 5 | 战后格局 | 战后利益分配 | 荆州归属、三分天下雏形 |

### 核心角色

| 角色 | 势力 | 核心特质 | 与玩家的初始关系 |
|------|------|---------|-----------------|
| 刘备 | 蜀汉 | 仁德、坚韧、知人善任 | 信任（+30） |
| 诸葛亮 | 蜀汉 | 智慧、谨慎、远见 | 中性（0），需建立信任 |
| 关羽 | 蜀汉 | 义气、骄傲、忠诚 | 略正面（+10） |
| 张飞 | 蜀汉 | 急躁、直率、勇猛 | 略正面（+5） |
| 曹操 | 曹魏 | 雄才、多疑、果断 | 敌对（-20） |
| 周瑜 | 东吴 | 骄傲、嫉才、有军事天赋 | 中性（0）偏警惕 |
| 鲁肃 | 东吴 | 温厚、亲和、主和派 | 中性（0）偏友善 |
| 孙权 | 东吴 | 年轻、犹豫、有大志 | 中性（0） |

### 验证目标

- [ ] 8 个角色的 AI 对话风格是否可区分
- [ ] 关系值变化是否影响 NPC 后续行为
- [ ] 记忆衰退后 NPC 是否"忘记"早期对话
- [ ] 玩家提醒机制是否能唤起 NPC "回忆"
- [ ] 跨章节的剧情一致性是否可维持

## 3. 故事包 B：福尔摩斯 · 波西米亚丑闻

### 基本信息

| 项目 | 内容 |
|------|------|
| ID | `sherlock-bohemia` |
| 题材 | 推理悬疑 |
| 原作 | 《波西米亚丑闻》（柯南·道尔），公有领域 |
| 截取范围 | 完整案件 |
| 玩家角色 | Watson 医生（或 Holmes 本人，可选） |
| 预计时长 | 1-2 小时 |

### 为什么选这个

- **交互模式完全不同**：调查→推理→行动，而非政治博弈
- **角色少而精**：验证小规模角色互动的质量
- **单案件结构**：验证短篇完整体验闭环
- **公有领域**：无版权风险
- **推理类对 AI 一致性要求最高**：是对系统的严格测试

### 章节规划

| 章 | 标题 | 核心事件 | 里程碑 |
|----|------|---------|--------|
| 1 | 委托 | 波西米亚国王来访，描述案件 | 接受委托 |
| 2 | 调查 | 调查 Irene Adler 的住所和习惯 | 发现关键线索 |
| 3 | 对决 | 执行计划取回照片 | 真相揭示、案件结局 |

### 核心角色

| 角色 | 核心特质 | 与玩家的初始关系 |
|------|---------|-----------------|
| Sherlock Holmes | 天才、傲慢、观察入微 | 搭档（+50，若玩家扮 Watson） |
| 波西米亚国王 | 焦虑、尊贵、隐藏信息 | 委托人（+20） |
| Irene Adler | 聪明、独立、优雅 | 未知（0）→ 随调查变化 |
| Mrs. Hudson | 温和、可靠、偶有抱怨 | 友善（+30） |

### 与故事包 A 的差异性

| 对比维度 | 三国（A） | 福尔摩斯（B） |
|---------|----------|-------------|
| 角色数量 | 8 | 4 |
| 章节数 | 5 | 3 |
| 交互风格 | 多方博弈 | 线性调查 |
| 关系复杂度 | 多对多 | 一对一为主 |
| 记忆需求 | 高（长篇多角色） | 中（线索记忆） |
| 知识边界重要性 | 中 | 极高（谁知道什么直接影响推理） |
| 世界规则复杂度 | 中（历史约束） | 低（维多利亚时代伦敦） |

### 验证目标

- [ ] 推理类场景 AI 的线索一致性
- [ ] 知识边界约束是否有效（NPC 不泄露不该知道的信息）
- [ ] 短篇完整体验闭环是否流畅
- [ ] 少角色场景下 AI 对话深度是否足够
- [ ] 案件结局是否因玩家行为产生有意义的差异

## 4. 两个故事包的协同验证

通过这两个差异性极大的故事包，可以验证以下系统层面的通用性：

| 验证项 | 通过标准 |
|--------|---------|
| Story Package 格式通用性 | 同一套 schema 可描述两种完全不同的故事 |
| AI 输出协议通用性 | TurnResult 结构可承载政治对话和推理调查 |
| 记忆系统通用性 | 记忆衰退对长篇多角色和短篇少角色都有效 |
| 存档系统通用性 | 存档/读档在两种故事类型下都能正确恢复状态 |
| 介入系统通用性 | 被动/主动/强制介入在不同叙事节奏下都感觉自然 |
| Prompt 模板系统 | 同一套模板框架可适配不同世界观和角色风格 |
