# 玩法蓝图 (Gameplay Blueprint)

> 版本：v0.1 | 状态：初版定稿

## 1. 核心体验目标

玩家应该感受到：

- **沉浸感**：我就在这个故事世界里，NPC 的反应是真实的
- **自由度**：我可以说任何话、做任何事，世界会合理回应
- **紧张感**：我的存档有限，我的选择有后果，NPC 会忘记过去的事
- **熟悉感**：我认识这些角色和世界，但这次的故事是我自己的

## 2. 核心循环 (Core Loop)

```
选择故事 → 进入章节 → [自动推进叙事 ↔ 玩家介入] → 里程碑推进 → 章节结束 → 下一章 / 结局
```

单次交互循环：

```
AI 输出一个 Beat → 自动展示给玩家 → 检查是否为介入点
  ├── 非介入点 → 延迟后自动推进下一个 Beat
  ├── 被动介入点 → 展示建议选项 + 输入框，等待玩家或超时
  ├── 主动介入（玩家按按钮） → 暂停，展示建议选项 + 输入框
  └── 强制介入点 → 暂停，必须操作
玩家输入 → AI 推演 → 生成下一批 Beat → 循环
```

## 3. 叙事节拍系统 (Story Beat System)

### Beat 定义

Beat 是叙事的最小推进单元，每个 Beat 对应一段完整的叙事片段。

### Beat 类型

| 类型 | 含义 | 典型长度 | 示例 |
|------|------|---------|------|
| `narration` | 旁白描写 | 2-4 句 | 场景描写、气氛渲染 |
| `dialogue` | NPC 对话 | 1-3 句 | 单个角色发言 |
| `event` | 事件发生 | 1-3 句 | 信使到来、战斗爆发 |
| `transition` | 场景转换 | 1 句 | 时间推移、地点切换 |

### Beat 自动推进规则

- 每个 Beat 展示后，等待 `auto_advance_delay` 时间后自动推进到下一个
- `auto_advance_delay` 基于 Beat 文本长度动态计算：`base_delay + char_count * per_char_delay`
- 玩家可随时暂停自动推进

### Beat 推进参数

| 参数 | 类型 | 初始值 | 说明 |
|------|------|--------|------|
| `base_delay_ms` | int | 500 | 每个 Beat 的基础展示延迟 |
| `per_char_delay_ms` | int | 50 | 每个字符额外增加的延迟 |
| `max_delay_ms` | int | 8000 | 单个 Beat 的最大延迟上限 |
| `player_speed_multiplier` | float | 1.0 | 玩家可调的速度倍率 (0.5x-3.0x) |

## 4. 介入系统 (Intervention System)

### 4.1 被动介入 (Reactive Intervention)

AI 在以下条件下自动生成介入点：

- NPC 对话直接向玩家提问
- 场景中出现需要玩家做决定的事件
- 每隔 `beats_between_reactive` 个 Beat 至少触发一次

**展示内容**：
- 3-4 个 AI 生成的建议选项（附简短行为基调标签）
- 一个自由文本输入框
- "继续旁观"按钮（触发沉默行为）

**超时行为**：
- `silence_timeout_ms` 后，若玩家无操作，按沉默处理
- 被动介入的超时时间较长（默认 30 秒）

### 4.2 主动介入 (Proactive Intervention)

- 在任何 Beat 展示期间，玩家可点击「我要行动」按钮
- 立即暂停当前自动推进
- AI 基于当前上下文生成 3-4 个建议选项
- 同时展示自由文本输入框

### 4.3 强制介入 (Forced Intervention)

- 由 Story Package 中的里程碑或关键 Beat 标记触发
- 故事完全暂停，必须等待玩家操作
- 无超时，不会自动跳过
- 视觉上有明显区分（如背景变化、提示文字）

### 介入系统参数

| 参数 | 类型 | 初始值 | 说明 |
|------|------|--------|------|
| `beats_between_reactive` | int | 4 | 两个被动介入点之间的最小 Beat 数 |
| `silence_timeout_ms` | int | 30000 | 被动介入点的等待超时 |
| `suggestions_count` | int | 3 | AI 生成的建议选项数量 |
| `suggestion_diversity` | enum | `balanced` | 建议多样性策略：conservative / balanced / diverse |

## 5. 沉默即行动 (Silence as Action)

### 规则定义

当玩家在被动介入点不做任何操作时：

1. 角色被视为"沉默旁观"
2. AI 在下一轮推演时，将"玩家沉默"作为输入的一部分
3. NPC 根据自身性格对沉默做出反应：
   - 强势角色可能追问或不满
   - 温和角色可能替你做决定
   - 敏锐角色可能解读你的沉默

### 沉默不适用于

- 强制介入点：不存在沉默选项
- 主动介入：玩家主动按了按钮，说明有行动意图

### 沉默的叙事价值

沉默不是"什么都没发生"，它是一种角色表达：

- 在谈判中沉默 → 可能被解读为犹豫或城府
- 在争吵中沉默 → 可能被解读为回避或忍让
- 在危机中沉默 → 可能被解读为镇定或恐惧

AI 应根据当前情境和角色性格选择合理的解读。

## 6. 记忆衰退系统 (Memory Decay)

### 分层模型

| 层级 | 名称 | 内容 | 进入/退出条件 |
|------|------|------|-------------|
| L1 | 鲜明记忆 | 最近 N 轮原始对话 | 新对话自动进入；超出窗口降级到 L2 |
| L2 | 模糊记忆 | 摘要压缩后的历史 | 从 L1 降级触发摘要生成；超出 token 预算降级到 L3 |
| L3 | 遗忘区 | 极简索引标签 | 从 L2 降级；默认不参与推理 |
| L4 | 永久记忆 | 里程碑、重大决策、核心关系 | 系统自动标记；永不衰退 |

### 参数

| 参数 | 类型 | 初始值 | 说明 |
|------|------|--------|------|
| `vivid_window_size` | int | 10 | 鲜明记忆保留的对话轮数 |
| `faded_budget_tokens` | int | 2000 | 模糊记忆的 token 预算上限 |
| `compression_trigger_turns` | int | 12 | 超过此轮数的鲜明记忆触发压缩 |
| `permanent_auto_tag` | list | `[milestone, betrayal, alliance, death]` | 自动进入永久记忆的事件类型 |

### 玩家提醒机制

玩家可以在自由输入中提及过去的事（如"你还记得在长坂坡……"），AI 从模糊记忆或遗忘区搜索相关内容，尝试"回忆"并做出反应。回忆可能成功也可能失败，取决于信息所在层级。

## 7. 存档系统 (Save System)

### 存档类型

| 类型 | 触发 | 消耗 | 保留数量 |
|------|------|------|---------|
| 自动锚点 | 里程碑完成 | 免费 | 每章最近 3 个 |
| 手动存档 | 玩家主动 | 1 存档点 | 不限（受存档点数量限制） |

### 存档点经济

| 参数 | 类型 | 初始值 | 说明 |
|------|------|--------|------|
| `save_points_per_chapter` | int | 3 | 每章开始时获得的存档点 |
| `bonus_save_conditions` | list | `[discover_secret, perfect_dialogue]` | 额外存档点获取条件 |
| `max_manual_saves` | int | 10 | 最大手动存档槽位数 |

### 读档行为

- 回到存档时的完整世界状态（包括记忆层级状态）
- 从存档点开始重新推演，AI 生成可能不同的结果
- 存档点之后发生的一切被丢弃

## 8. AI 建议选项生成规则

### 质量要求

- **多样性**：3-4 个选项覆盖至少 2 种不同的行为方向
- **信息量**：每个选项附带简短标签（如"谨慎""激进""外交"），不剧透后果
- **相关性**：选项基于当前场景、角色关系、已有伏笔生成
- **不封死自由度**：自由输入框始终可用

### 选项生成的上下文输入

- 当前场景描述
- 活跃 NPC 列表及关系状态
- 最近的记忆上下文
- 当前章节的叙事基调

## 9. 原型验证清单

以下假设需要在原型阶段验证：

| # | 假设 | 验证方式 | 成功标准 |
|---|------|---------|---------|
| 1 | 自动推进 + 我要行动的节奏舒适 | 5 人内测，记录介入频率 | 平均每 3-6 个 Beat 介入一次 |
| 2 | AI 建议选项降低了输入门槛 | 统计选项使用 vs 自由输入比例 | 选项使用率 > 40% |
| 3 | 沉默机制被理解且被利用 | 观察玩家是否主动选择沉默 | 至少 20% 场景中有玩家沉默 |
| 4 | 记忆衰退增加了叙事趣味 | 内测反馈问卷 | 正面评价 > 60% |
| 5 | 限量存档增加了紧张感 | 记录存档使用时机 | 存档点不在第一时间全部用完 |
